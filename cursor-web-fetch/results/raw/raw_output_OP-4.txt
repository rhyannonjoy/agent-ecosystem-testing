## Create a MongoDB Search Index

a MongoDB Search index is a data structure that categorizes data in an easily searchable format. It maps terms with documents that contain those terms to enable fast retrieval of documents at query-time.

You must configure a MongoDB Search index to query data using MongoDB Search. We recommend that you index the fields that you regularly use to sort or filter your data.

To learn more, see [Supported Clients](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/manage-indexes/#std-label-fts-manage-indexes).

In this section, you create a MongoDB Search index on the sample [movies](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/sample-data/sample-mflix/#std-label-mflix-movies) collection. You can create the MongoDB Search index in either an Atlas cluster or a deployment hosted on your local computer.

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

This query has the following search criteria:

- The `plot` field must contain the word `baseball`.

Paste the following query into the Query Editor, and then click the Search button in the Query Editor.

```json
[
  {
    $search:

          path: "plot"
      }
```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

This query has the following search criteria:

- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

Paste the following query into the Query Editor, and then click the Search button in the Query Editor.

```json
[
  {
    "$search": {
      "compound": {
        "must": [
          {
            "text": {
              "query": "baseball",
              "path": "plot"
            }
          }
        ],
        "mustNot": [
          {
            "text": {
              "query": ["Comedy", "Romance"],
              "path": "genres"
            }
          }
        ]
      }
    }
  }
]
```


## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

This query has the following search criteria:

- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

Copy and paste the following query into the Query Editor, and then click the Search button in the Query Editor.

```json
[
  {
    "$search": {
      "compound": {
        "must": [
          {
            "text": {
              "query": "baseball",
              "path": "plot"
            }
          }
        ],
        "mustNot": [
          {
            "text": {
              "query": ["Comedy", "Romance"],
              "path": "genres"
            }
          }
        ]
      },
      "sort": {
        "released": -1
      }
    }
  }
]

```

## Continue Learning

### Run Autocomplete and Partial Match Queries

You can use MongoDB Search to run autocomplete and partial queries that return results for partial words. This is useful to retrieve results with increasing accuracy as more characters are entered in your application's search field. You must index the fields as the MongoDB Search [autocomplete](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/field-types/autocomplete-type/#std-label-bson-data-types-autocomplete) type to use the [autocomplete](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/autocomplete/#std-label-autocomplete-ref) operator.

To get started, see [How to Run Autocomplete and Partial Match MongoDB Search Queries](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/tutorial/partial-match/#std-label-partial-match-tutorial).

### Group Search Results with Facets

You can use MongoDB Search to run facet queries that group search results by string, date, or numeric values. You must create an index with a [facet definition](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/facet/#std-label-fts-facet-definition) to use the [`facet` (MongoDB Search Operator)](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/facet/#std-label-fts-facet-ref) collector.

To get started, see [How to Use Facets with MongoDB Search](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/tutorial/facet-tutorial/#std-label-facet-tutorial).

## Create the MongoDB Search index.

- Connect to the cluster using [`mongosh`](https://www.mongodb.com/docs/mongodb-shell/#mongodb-binary-bin.mongosh).

Open [`mongosh`](https://www.mongodb.com/docs/mongodb-shell/#mongodb-binary-bin.mongosh) in a terminal window and connect to your Atlas cluster. For detailed instructions on connecting, see [Connect via mongosh](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/mongo-shell-connection/#std-label-connect-mongo-shell).

```shell

```

```shell
switched to db sample_mflix

```

- Run the [`db.collection.createSearchIndex()`](https://www.mongodb.com/docs/manual/reference/method/db.collection.createSearchIndex/#mongodb-method-db.collection.createSearchIndex) method to create the index.

```shell
db.movies.createSearchIndex(
   "default",
   { mappings: { dynamic: true } }
)

```

Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.

```javascript
db.movies.aggregate([
```

```javascript
        "mustNot": [
          {
            "text": {

            }

  },

  }
])
```

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

```csharp
using MongoDB.Driver;

public class BasicQuery
{
    private const string MongoConnectionString = "<connection-string>";

    {
        // allow automapping of the camelCase database fields to our MovieDocument
        var camelCaseConvention = new ConventionPack { new CamelCaseElementNameConvention() };
        ConventionRegistry.Register("CamelCase", camelCaseConvention, type => true);

        // connect to your Atlas cluster
        var mongoClient = new MongoClient(MongoConnectionString);
```

## Refine your search.

- Modify your `Program.cs` file with the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.

```csharp
using MongoDB.Bson;
using MongoDB.Bson.Serialization.Attributes;
using MongoDB.Bson.Serialization.Conventions;
using MongoDB.Driver;
using MongoDB.Driver.Search;

public class CompoundQuery
{
    private const string MongoConnectionString = "<connection-string>";

    public static void Main(string[] args)
    {
        // allow automapping of the camelCase database fields to our MovieDocument
        var camelCaseConvention = new ConventionPack { new CamelCaseElementNameConvention() };

        // connect to your Atlas cluster

        var mflixDatabase = mongoClient.GetDatabase("sample_mflix");
        var moviesCollection = mflixDatabase.GetCollection<MovieDocument>("movies");

        // declare data for the compound query
        string[] genres = { "Comedy", "Romance" };

        // define and run pipeline
        var results = moviesCollection.Aggregate()
            .Search(Builders<MovieDocument>.Search.Compound()
                .Must(Builders<MovieDocument>.Search.Text(movie => movie.Plot, "baseball"))
                .MustNot((Builders<MovieDocument>.Search.Text(movie => movie.Genres, genres))))
            .Project<MovieDocument>(Builders<MovieDocument>.Projection
                .Exclude(movie => movie.Id)
                .Include(movie => movie.Plot)
                .Include(movie => movie.Title)
                .Include(movie => movie.Genres))

            .ToList();

        }
```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

- Modify your `Program.cs` file with the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```csharp
using MongoDB.Bson;
using MongoDB.Bson.Serialization.Attributes;
using MongoDB.Bson.Serialization.Conventions;
using MongoDB.Driver;
using MongoDB.Driver.Search;

public class SortQuery
{
    private const string MongoConnectionString = "<connection-string>";

    public static void Main(string[] args)
    {
        // allow automapping of the camelCase database fields to our MovieDocument
        var camelCaseConvention = new ConventionPack { new CamelCaseElementNameConvention() };
        ConventionRegistry.Register("CamelCase", camelCaseConvention, type => true);

        // connect to your Atlas cluster
        var mongoClient = new MongoClient(MongoConnectionString);
        var mflixDatabase = mongoClient.GetDatabase("sample_mflix");
        var moviesCollection = mflixDatabase.GetCollection<MovieDocument>("movies");

        // declare data for the compound query
        string[] genres = { "Comedy", "Romance" };

        // define search options
        var searchOptions = new SearchOptions<MovieDocument>()
            {
                Sort = Builders<MovieDocument>.Sort.Descending(movie => movie.Released),
            };

        // define and run pipeline
        var results = moviesCollection.Aggregate()
            .Search(Builders<MovieDocument>.Search.Compound()
                .Must(Builders<MovieDocument>.Search.Text(movie => movie.Plot, "baseball"))
                .MustNot((Builders<MovieDocument>.Search.Text(movie => movie.Genres, genres))), searchOptions)
            .Project<MovieDocument>(Builders<MovieDocument>.Projection
                .Exclude(movie => movie.Id)
                .Include(movie => movie.Plot)
                .Include(movie => movie.Title)
                .Include(movie => movie.Genres)
                .Include(movie => movie.Released))
            .Limit(3)
            .ToList();

        // print results
        foreach (var movie in results)
        {
            Console.WriteLine(movie.ToJson());
        }
    }
}
```

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

- Create a new file named `run-query.go` and paste the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title` and `plot` fields.
- The `plot` field must contain the word `baseball`.

```go
package main

import (
	"context"
	"fmt"
	"time"

	"go.mongodb.org/mongo-driver/v2/bson"
	"go.mongodb.org/mongo-driver/v2/mongo"
	"go.mongodb.org/mongo-driver/v2/mongo/options"
)

func main() {
	var err error
	// connect to the Atlas cluster
	ctx := context.Background()
	client, err := mongo.Connect(options.Client().SetTimeout(5 * time.Second).ApplyURI("<connection-string>"))
	if err != nil {
		panic(err)
	}
	defer client.Disconnect(ctx)
	// set namespace
	collection := client.Database("sample_mflix").Collection("movies")
	// define pipeline
	searchStage := bson.D{{"$search", bson.D{
		{"text", bson.D{
			{"path", "plot"},
			{"query", "baseball"},
		}},
	}}}
	limitStage := bson.D{{"$limit", 3}}
	projectStage := bson.D{{"$project", bson.D{{"title", 1}, {"plot", 1}, {"_id", 0}}}}
	// run pipeline
	cursor, err := collection.Aggregate(ctx, mongo.Pipeline{searchStage, limitStage, projectStage})
	if err != nil {
		panic(err)
	}
	// print results
	var results []bson.D
	if err = cursor.All(context.TODO(), &results); err != nil {
		panic(err)
	}
	for _, result := range results {
		fmt.Println(result)
	}
}


```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

- Modify `run-query.go` to use the compound query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```go
package main

import (
	"context"
	"fmt"
	"time"

	"go.mongodb.org/mongo-driver/v2/bson"
	"go.mongodb.org/mongo-driver/v2/mongo"
	"go.mongodb.org/mongo-driver/v2/mongo/options"
)

func main() {
	var err error
	// connect to the Atlas cluster
	ctx := context.Background()
	client, err := mongo.Connect(options.Client().SetTimeout(5 * time.Second).ApplyURI("<connection-string>"))
	if err != nil {
		panic(err)
	}
	defer client.Disconnect(ctx)
	// set namespace
	collection := client.Database("sample_mflix").Collection("movies")
	// define pipeline
	searchStage := bson.D{{"$search", bson.M{
		"compound": bson.M{
			"must": bson.A{
				bson.M{
					"text": bson.D{
						{"path", "plot"}, {"query", "baseball"},
					},
				},
			},
			"mustNot": bson.A{
				bson.M{
					"text": bson.M{
						"path": "genres", "query": []string{"Comedy", "Romance"},
					},
				},
			},
		},
	}}}
	limitStage := bson.D{{"$limit", 3}}
	projectStage := bson.D{{"$project", bson.D{{"title", 1}, {"plot", 1}, {"genres", 1}, {"_id", 0}}}}
	// run pipeline
	cursor, err := collection.Aggregate(ctx, mongo.Pipeline{searchStage, limitStage, projectStage})
	if err != nil {
		panic(err)
	}
	// print results
	var results []bson.D
	if err = cursor.All(context.TODO(), &results); err != nil {
		panic(err)
	}
	for _, result := range results {
		fmt.Println(result)
	}
}
```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

- Modify `run-query.go` to add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```go
package main

import (
	"context"
	"fmt"
	"time"

	"go.mongodb.org/mongo-driver/v2/bson"
	"go.mongodb.org/mongo-driver/v2/mongo"
	"go.mongodb.org/mongo-driver/v2/mongo/options"
)

func main() {
	var err error
	// connect to the Atlas cluster
	ctx := context.Background()
	client, err := mongo.Connect(options.Client().SetTimeout(5 * time.Second).ApplyURI("<connection-string>"))
	if err != nil {
		panic(err)
	}
	defer client.Disconnect(ctx)
	// set namespace
	collection := client.Database("sample_mflix").Collection("movies")
	// define pipeline
	searchStage := bson.D{{"$search", bson.M{
		"compound": bson.M{
			"must": bson.A{
				bson.M{
					"text": bson.D{
						{"path", "plot"}, {"query", "baseball"},
					},
				},
			},
			"mustNot": bson.A{
				bson.M{
					"text": bson.M{
						"path": "genres", "query": []string{"Comedy", "Romance"},
					},
				},
			},
		},
		"sort": bson.D{
			{"released", -1},
		},
	}}}
	limitStage := bson.D{{"$limit", 3}}
	projectStage := bson.D{{"$project", bson.D{{"_id", 0}, {"title", 1}, {"plot", 1}, {"genres", 1}, {"released", 1}}}}
	// run pipeline
	cursor, err := collection.Aggregate(ctx, mongo.Pipeline{searchStage, limitStage, projectStage})
	if err != nil {
		panic(err)
	}
	// print results
	var results []bson.D
	if err = cursor.All(context.TODO(), &results); err != nil {
```

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

- Create a new file named `RunQuery.java` and paste the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title` and `plot` fields.
- The `plot` field must contain the word `baseball`.

```java
import java.util.Arrays;
import java.util.List;
import static com.mongodb.client.model.Aggregates.limit;
import static com.mongodb.client.model.Aggregates.project;
import static com.mongodb.client.model.Projections.excludeId;
import static com.mongodb.client.model.Projections.fields;
import static com.mongodb.client.model.Projections.include;
import com.mongodb.client.MongoClient;
import com.mongodb.client.MongoClients;
import com.mongodb.client.MongoCollection;
import com.mongodb.client.MongoDatabase;
import org.bson.Document;

public class RunQuery {
  public static void main( String[] args ) {
    String uri = "<connection-string>";

    try (MongoClient mongoClient = MongoClients.create(uri)) {
      MongoDatabase database = mongoClient.getDatabase("sample_mflix");
      MongoCollection<Document> collection = database.getCollection("movies");

      Document textQuery = new Document("query", "baseball").append("path","plot");

      Document searchStage = new Document("$search",
          new Document("index", "default")
              .append("text", textQuery));

      collection.aggregate(Arrays.asList(
          searchStage,
          limit(3),
          project(fields(excludeId(), include("title", "plot"))))
      ).forEach(doc -> System.out.println(doc.toJson()));	
    }
  }
}

```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

- Modify `RunQuery.java` to use the compound query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```java
import java.util.Arrays;
import java.util.List;
import static com.mongodb.client.model.Aggregates.limit;
import static com.mongodb.client.model.Aggregates.project;
import static com.mongodb.client.model.Projections.excludeId;
import static com.mongodb.client.model.Projections.fields;
import static com.mongodb.client.model.Projections.include;
import com.mongodb.client.MongoClient;
import com.mongodb.client.MongoClients;
import com.mongodb.client.MongoCollection;
import com.mongodb.client.MongoDatabase;
import org.bson.Document;

public class RunQuery {
  public static void main( String[] args ) {
    String uri = "<connection-string>";

    try (MongoClient mongoClient = MongoClients.create(uri)) {
      MongoDatabase database = mongoClient.getDatabase("sample_mflix");
      MongoCollection<Document> collection = database.getCollection("movies");
					
      Document compound = new Document("must", Arrays.asList(
          new Document("text", new Document("query", "baseball").append("path", "plot"))))
            .append("mustNot", Arrays.asList(
          new Document("text", new Document("query", Arrays.asList("Comedy", "Romance")).append("path", "genres"))));

      Document searchStage = new Document("$search",
          new Document("index", "default")
              .append("compound", compound));

      collection.aggregate(Arrays.asList(
          searchStage,
          limit(3),
          project(fields(excludeId(), include("title", "plot", "genres"))))
      ).forEach(doc -> System.out.println(doc.toJson()));	
    }
  }
}
```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. - Modify `RunQuery.java` to add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```java
import static com.mongodb.client.model.Projections.fields;
import static com.mongodb.client.model.Projections.include;
import com.mongodb.client.MongoClient;
import com.mongodb.client.MongoClients;
import com.mongodb.client.MongoCollection;
import com.mongodb.client.MongoDatabase;
import org.bson.Document;

public class RunQuery {
  public static void main( String[] args ) {
    String uri = "<connection-string>";

    try (MongoClient mongoClient = MongoClients.create(uri)) {
      MongoDatabase database = mongoClient.getDatabase("sample_mflix");

      Document compound = new Document("must", Arrays.asList(
          new Document("text", new Document("query", "baseball").append("path", "plot"))))
            .append("mustNot", Arrays.asList(
          new Document("text", new Document("query", Arrays.asList("Comedy", "Romance")).append("path", "genres"))));

      Document searchStage = new Document("$search",
          new Document("index", "default")
              .append("compound", compound)
              .append("sort", new Document("released", -1)));

      collection.aggregate(Arrays.asList(
          searchStage,
          limit(3),
          project(fields(excludeId(), include("title", "plot", "genres", "released"))))
```

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

- Create a new file named `RunQuery.kt` and paste the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title` and `plot` fields.
- The `plot` field must contain the word `baseball`.

```kotlin
import com.mongodb.client.model.Aggregates.limit
import com.mongodb.client.model.Aggregates.project
import com.mongodb.client.model.Filters.eq
import com.mongodb.client.model.Projections.*
import com.mongodb.kotlin.client.coroutine.MongoClient
import kotlinx.coroutines.runBlocking
import org.bson.Document

fun main() {
    // establish connection and set namespace
    val uri = "<connection-string>"
    val mongoClient = MongoClient.create(uri)
    val database = mongoClient.getDatabase("sample_mflix")
    val collection = database.getCollection<Document>("movies")

    runBlocking {
        // define query
        val agg = Document("query", "baseball").append("path","plot")

        // run query and print results
        val resultsFlow = collection.aggregate<Document>(
            listOf(
                eq("\$search", eq("text", agg)),
                limit(3),
                project(fields(excludeId(), include("title", "plot")))
            )
        )
        resultsFlow.collect { println(it) }
    }
    mongoClient.close()
}

```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

- Modify `RunQuery.kt` to use the compound query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```kotlin
import com.mongodb.client.model.Aggregates.limit
import com.mongodb.client.model.Aggregates.project
import com.mongodb.client.model.Filters.eq
import com.mongodb.client.model.Projections.*
import com.mongodb.kotlin.client.coroutine.MongoClient
import kotlinx.coroutines.runBlocking
import org.bson.Document

fun main() {
    // establish connection and set namespace
    val uri = "<connection-string>"
    val mongoClient = MongoClient.create(uri)
    val database = mongoClient.getDatabase("sample_mflix")
    val collection = database.getCollection<Document>("movies")

    runBlocking {
        // define query
        val compoundQuery = Document(
            "must", listOf(
                Document("text", Document("query", "baseball")
                    .append("path", "plot"))
            )
        )
            .append(
                "mustNot", listOf(
                    Document("text", Document("query", listOf("Comedy", "Romance"))
                        .append("path", "genres"))
                )
            )

        // run query and print results
        val resultsFlow = collection.aggregate<Document>(
            listOf(
                eq("\$search", eq("compound", compoundQuery)),
                limit(3),
                project(fields(
                    excludeId(),
                    include("title", "plot", "genres")
                ))
            )
        )
        resultsFlow.collect { println(it) }
    }
    mongoClient.close()
}
```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

- Modify `RunQuery.kt` to add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```kotlin
import com.mongodb.client.model.Aggregates.limit

    val database = mongoClient.getDatabase("sample_mflix")
    val collection = database.getCollection<Document>("movies")

    runBlocking {

            "must", listOf(

            )
        )
            .append(
                "mustNot", listOf(
                    Document("text", Document("query", listOf("Comedy", "Romance"))
                        .append("path", "genres"))

            listOf(
                Document("\$search", Document("compound", compoundQuery).append("sort", sortConfig)),
                limit(3),
                project(fields(
                    excludeId(),
                    include("title", "plot", "genres", "released")
                ))
            )
        )

    }
```

## Create a MongoDB Search Index

a MongoDB Search index is a data structure that categorizes data in an easily searchable format. It maps terms with documents that contain those terms to enable fast retrieval of documents at query-time.

You must configure a MongoDB Search index to query data using MongoDB Search. We recommend that you index the fields that you regularly use to sort or filter your data.

To learn more, see [Supported Clients](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/manage-indexes/#std-label-fts-manage-indexes).

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

- Create a new file named `run-query.js` and paste the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title` and `plot` fields.
- The `plot` field must contain the word `baseball`.

```javascript
const { MongoClient } = require("mongodb");

async function main() {
  // Replace the placeholder with your connection string
  const uri = "<connection-string>";
  const client = new MongoClient(uri);

  try {
    await client.connect();
    const database = client.db("sample_mflix");
    const movies = database.collection("movies");

    const query = [
      {
        $search:
        {
          text: {
            query: "baseball",
            path: "plot",
          },
        }
      },
      {
        $limit: 3,
      },
      {
        $project: {
          _id: 0,
          title: 1,
          plot: 1,
        },
      },
    ];

    const cursor = movies.aggregate(query);
    await cursor.forEach(doc => console.log(doc));
  } finally {
    await client.close();
  }
}

main().catch(console.error);


```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

- Modify `run-query.js` to use the compound query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```javascript
const { MongoClient } = require("mongodb");

async function main() {
  // Replace the placeholder with your connection string
  const uri = "<connection-string>";
  const client = new MongoClient(uri);

  try {
    await client.connect();
    const database = client.db("sample_mflix");
    const movies = database.collection("movies");

    const query = [
      {
        $search: {
          compound: {
            must: [
              {
                text: {
                  query: "baseball",
                  path: "plot",
                }
              }
            ],
            mustNot: [
              {
                text: {
                  query: ["Comedy", "Romance"],
                  path: "genres",
                },
              }
            ]
          }
        }
      },
      {
        $limit: 3
      },
      {
        $project: {
          _id: 0,
          title: 1,
          plot: 1,
          genres: 1
        }
      }
    ];

    const cursor = movies.aggregate(query);
    await cursor.forEach(doc => console.log(doc));
  } finally {
    await client.close();
  }
}

main().catch(console.error);


```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

- Modify `run-query.js` to add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```javascript
const { MongoClient } = require("mongodb");

async function main() {
  // Replace the placeholder with your connection string
  const uri = "<connection-string>";
  const client = new MongoClient(uri);

  try {
    await client.connect();
    const database = client.db("sample_mflix");
    const movies = database.collection("movies");

    const query = [
      {
        $search: {
          compound: {
            must: [
              {
                text: {
                  query: "baseball",
                  path: "plot",
                }
              }
            ],
            mustNot: [
              {
                text: {
                  query: ["Comedy", "Romance"],
                  path: "genres",
                }
              }
            ]
          },
          sort: {
            released: -1
          }
        }
      },
      {
        $limit: 3
      },
      {
        $project: {
          _id: 0,
          title: 1,
          plot: 1,
          genres: 1,
          released: 1
        }
      }
    ];

    const cursor = movies.aggregate(query);
    await cursor.forEach(doc => console.log(doc));
  } finally {
    await client.close();
  }
}

main().catch(console.error);


```

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

- Create a new file named `run_query.py` and paste the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title` and `plot` fields.
- The `plot` field must contain the word `baseball`.

```python
import pymongo

client = pymongo.MongoClient('<connection-string>')
result = client['sample_mflix']['movies'].aggregate([
  {
    '$search':
     {
        'text': {
          'query': 'baseball',
          'path': 'plot'
        }
     }
  },
  {
    '$limit': 3
  },
  {
    '$project': {
       '_id': 0,
       'title': 1,
       'plot': 1
    }
  }
])

for i in result:
    print(i)


```

- Specify the ` `, then run the query:

```bash
python run_query.py

```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

- Modify `run_query.py` to use the compound query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```python
import pymongo

client = pymongo.MongoClient('<connection-string>')
result = client['sample_mflix']['movies'].aggregate([
  {
    '$search': {
      'compound': {
        'must': [
          {
            'text': {
              'query': 'baseball',
              'path': 'plot'
            }
          }
        ],
        'mustNot': [
          {
            'text': {
              'query': ['Comedy', 'Romance'],
              'path': 'genres'
            }
          }
        ]
      }
    }
  },
  {
    '$limit': 3
  },
  {
    '$project': {
      '_id': 0,
      'title': 1,
      'plot': 1,
      'genres': 1
    }
  }
])

for i in result:
    print(i)


```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

- Modify `run_query.py` to add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```python
import pymongo

client = pymongo.MongoClient('<connection-string>')
result = client['sample_mflix']['movies'].aggregate([
  {
    '$search': {
      'compound': {
        'must': [
          {
            'text': {
              'query': 'baseball',
              'path': 'plot'
            }
          }
        ],
        'mustNot': [
          {
            'text': {
              'query': ['Comedy', 'Romance'],
              'path': 'genres'
            }
          }
        ]
      },
      'sort': {
          'released': -1
      }
    }
  },
  {
    '$limit': 3
  },
  {
    '$project': {
      '_id': 0,
      'title': 1,
      'plot': 1,
      'genres': 1,
      'released': 1
    }
  }
])

for i in result:
    print(i)


```

```bash
python run_query.py
```

## Create the MongoDB Search index.

- Initialize your Rust project.

```
# Create a new Rust project
cargo new atlas-search-quickstart && cd atlas-search-quickstart

```

Add the following dependencies to your `Cargo.toml` file:

```
[dependencies]
  serde = "1.0"
  futures = "0.3"
  tokio = {version = "1.0", features = ["full"]}

[dependencies.mongodb]
  version = "3.0"

```

For detailed installation instructions, see the [MongoDB Rust Driver Download and Install](https://www.mongodb.com/docs/drivers/rust/current/quick-start/download-and-install/#std-label-rust-quick-start-download-and-install) guide.

- Define the index.

Paste the following code into `src/main.rs`.

```rust
use mongodb::bson::{doc, Document};
use mongodb::{Client, Collection, SearchIndexModel, SearchIndexType};

#[tokio::main]
async fn main() -> mongodb::error::Result<()> {
    // Connects to your Atlas deployment
    let uri =
        "<connection-string>";

    let client = Client::with_uri_str(uri).await?;

    // Sets the namespace
    let collection: Collection<Document> = client
        .database("sample_mflix")
        .collection("movies");

    // Defines your MongoDB Search index
    let index = doc! {
        "mappings": doc! {
            "dynamic": true
        }
    };

    let idx_model = SearchIndexModel::builder()
        .definition(index)
        .name("search_idx".to_string())
        .index_type(SearchIndexType::Search)
        .build();

    // Runs the helper method
    let result = collection.create_search_index(idx_model).await?;
    println!("{}", result);

    Ok(())
}


```

Replace ` ` with the connection string for your Atlas cluster or local Atlas deployment.

Your connection string should use the following format:

```
mongodb+srv://<db_username>:<db_password>@<clusterName>.<hostname>.mongodb.net

```

To learn more, see [Connect to a Cluster via Client Libraries](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/driver-connection/#std-label-connect-via-driver).

Your connection string should use the following format:

```
mongodb://localhost:<port-number>/?directConnection=true
```

## Build a MongoDB Search Query

MongoDB Search queries take the form of an [aggregation pipeline stage](https://www.mongodb.com/docs/manual/aggregation/). You use MongoDB Search primarily with the [`$search`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/search/#mongodb-pipeline-pipe.-search) stage, which must be the first stage in the query pipeline. You can also use this stage in conjunction with other stages in your pipeline.

When you run a MongoDB Search query, MongoDB Search uses the search index to locate and retrieve relevant data from the collection. MongoDB Search also provides the [`$searchMeta`](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/aggregation-stages/searchMeta/#mongodb-pipeline-pipe.-searchMeta) stage, multiple sub-pipelines, and several query [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-operators-ref) and [collectors](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-collectors-ref) that you can use to further refine your search results.

To learn more, see [Queries and Indexes](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/searching/#std-label-fts-about-queries).

In this section, you run queries against the indexed collection.

## Run a basic query.

Run a basic `$search` query by using the [text](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/text/#std-label-text-ref) operator.

- Create a new file named `src/main.rs` and paste the following code.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title` and `plot` fields.
- The `plot` field must contain the word `baseball`.

```rust
use mongodb::bson::{doc, DateTime};
use mongodb::{Client, Collection};
use serde::{Deserialize, Serialize};
use futures::TryStreamExt;

#[derive(Debug, Deserialize, Serialize)]
struct Movie {
    title: Option<String>,
    plot: Option<String>,
    genres: Option<Vec<String>>,
    released: Option<DateTime>,
}

#[tokio::main]
async fn main() -> mongodb::error::Result<()> {
    // Replace the placeholder with your connection string
    let uri = "<connection-string>";
    let client = Client::with_uri_str(uri).await?;

    let collection: Collection<Movie> = client
        .database("sample_mflix")
        .collection("movies");

    let query = vec![
        doc! {
            "$search": {
                "index": "search_idx",
                "text": {
                    "query": "baseball",
                    "path": "plot"
                }
            }
        },
        doc! {
            "$limit": 3
        },
        doc! {
            "$project": {
                "_id": 0,
                "title": 1,
                "plot": 1
            }
        }
    ];

    println!("Executing aggregation query...");
    let mut results = movies.aggregate(query).await?;

    println!("Processing results...");
    while let Some(doc) = results.try_next().await? {
        let movie: Movie = mongodb::bson::from_document(doc)?;
        println!("{{ \"title\": {:?}, \"plot\": {:?} }}",
                 movie.title.unwrap_or_else(|| "N/A".to_string()),
                 movie.plot.unwrap_or_else(|| "N/A".to_string()));
    }
    Ok(())
}


```

## Refine your search.

MongoDB Search provides several [operators](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-and-collectors/#std-label-fts-operators) that you can use to build complex queries and retrieve more specific results for your use case. Update your `$search` query to use the [compound](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/operators-collectors/compound/#std-label-compound-ref) operator to combine multiple operators into a single query.

- Modify `src/main.rs` to use the compound query.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, and `genres` fields.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```rust
use mongodb::bson::{doc, DateTime, from_document};
use mongodb::{Client, Collection};
use serde::{Deserialize, Serialize};
use futures::TryStreamExt;

#[derive(Debug, Deserialize, Serialize)]
struct Movie {
    title: Option<String>,
    plot: Option<String>,
    genres: Option<Vec<String>>,
    released: Option<DateTime>,
}

#[tokio::main]
async fn main() -> mongodb::error::Result<()> {
    // Replace the placeholder with your connection string
    let uri = "<connection-string>";
    let client = Client::with_uri_str(uri).await?;

    let query = vec![
        doc! {
            "$search": {
                "index": "search_idx",
                "compound": {
                    "must": [
                        {
                            "text": {
                                "query": "baseball",
                                "path": "plot"
                            }
                        }
                    ],
                    "mustNot": [
                        {
                            "text": {
                                "query": ["Comedy", "Romance"],
                                "path": "genres"
                            }
                        }
                    ]
                }
            }
        },
        doc! {
            "$limit": 3
        },
        doc! {
            "$project": {
                "_id": 0,
                "title": 1,
                "plot": 1,
                "genres": 1
            }
        }
    ];

    let movies: Collection<Movie> = client
        .database("sample_mflix")
        .collection("movies");

    let mut result = collection.aggregate(query).await?;

    while let Some(doc) = result.try_next().await? {
        let movie: Movie = from_document(doc)?;
        println!("  Title: '{}'", movie.title.unwrap_or_else(|| "N/A".to_string()));
        println!("  Plot: '{}'", movie.plot.unwrap_or_else(|| "N/A".to_string()));
        println!("  Genres: {:?}", movie.genres.unwrap_or_default());
        println!();
    }
    Ok(())
}
```

## Process your results.

MongoDB Search provides several [search options](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/search-options/#std-label-fts-search-options) that you can use to further process your MongoDB Search query results. Add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option to your query to display the results in a specific order.

- Modify `src/main.rs` to add the [sort](https://mongodbcom-cdn.staging.corp.mongodb.com/docs/atlas/atlas-search/sort/#std-label-sort-ref) option.

This query has the following search criteria:

- [$limit](https://www.mongodb.com/docs/manual/reference/operator/aggregation/limit/) and [$ project](https://www.mongodb.com/docs/manual/reference/operator/aggregation/project/) stages to return only 3 documents and the `title`, `plot`, `genres`, and `released` fields.
- Sort the results by the `released` date field in descending order. MongoDB Search returns the most recent movies first.
- The `genres` field must not contain either `Comedy` or `Romance`.
- The `plot` field must contain the word `baseball`.

```rust
use mongodb::bson::{doc, DateTime, from_document};
use mongodb::{Client, Collection};
use serde::{Deserialize, Serialize};
use futures::TryStreamExt;

#[derive(Debug, Deserialize, Serialize)]
struct Movie {
    title: Option<String>,
    plot: Option<String>,
    genres: Option<Vec<String>>,
    released: Option<DateTime>,
}

#[tokio::main]
async fn main() -> mongodb::error::Result<()> {
    // Replace the placeholder with your connection string
    let uri = "<connection-string>";
    let client = Client::with_uri_str(uri).await?;

    let query = vec![
        doc! {
            "$search": {
                "index": "search_idx",
                "compound": {
                    "must": [
                        {
                            "text": {
                                "query": "baseball",
                                "path": "plot"
                            }
                        }
                    ],
                    "mustNot": [
                        {
                            "text": {
                                "query": ["Comedy", "Romance"],
                                "path": "genres"
                            }
                        }
                    ]
                },
                "sort": {
                    "released": -1
                }
            }
        },
        doc! {
            "$limit": 3
        },
        doc! {
            "$project": {
                "_id": 0,
                "title": 1,
                "plot": 1,
                "genres": 1,
                "released": 1
            }
        }
    ];

    let movies: Collection<Movie> = client
        .database("sample_mflix")
        .collection("movies");
    let mut cursor = movies.aggregate(query).await?;
    while let Some(doc) = cursor.try_next().await? {
        let movie: Movie = from_document(doc)?;
        println!("Title: {:?}", movie.title.unwrap_or_else(|| "N/A".to_string()));
        println!("Plot: {:?}", movie.plot.unwrap_or_else(|| "N/A".to_string()));
        println!("Genres: {:?}", movie.genres.unwrap_or_default());
        if let Some(released) = movie.released {
            println!("Released: {:?}", released);
        }
        println!();
    }

    Ok(())
}
```

```shell
else
echo "Sample data not found. Running mongorestore..."
if [ -f "/sampledata.archive" ]; then
   mongorestore --archive=/sampledata.archive
   echo "Sample data restored successfully."
else
   echo "Warning: sampledata.archive not found"
fi
fi

echo "MongoDB initialization completed."

```

### Set up the Docker environment

To create the Docker network, run the following command:

```shell
docker network create search-community

```

### Start `mongod` and `mongot`

Use the following steps to launch both services:

- Launch `mongod` and `mongot`:

```shell
docker compose up -d

```

- Wait for both services to fully initialize.

To monitor the logs of both services, run the following command:

```shell
docker compose logs -f

```

The initialization is complete when the logs return `MongoDB initialization completed`.
