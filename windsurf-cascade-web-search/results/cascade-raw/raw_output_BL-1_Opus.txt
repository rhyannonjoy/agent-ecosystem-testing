# create Event

[Chunk 50 - Summary]
## Summary

```
create
```

New in version 6.0.
A create event occurs when a collection is created on a watched database and the change stream has the [showExpandedEvents](https://www.mongodb.com/docs/manual/reference/change-events/) option set to true.

```
create
```

```
true
```

[Chunk 51 - Description]
## Description

_id

```
_id
```

Document
A BSON object, which serves as an identifier for the change stream event. This value is used as the resumeToken for the resumeAfter parameter when resuming a change stream. The fields within the _id object depend on the MongoDB versions and, in some cases, the [feature compatibility version (FCV)](https://www.mongodb.com/docs/manual/reference/command/setFeatureCompatibilityVersion/) at the time of the change stream's opening or resumption.

```
resumeToken
```

```
resumeAfter
```

```
_id
```

For an example of resuming a change stream by resumeToken, see [Resume a Change Stream.](https://www.mongodb.com/docs/manual/changeStreams/)

```
resumeToken
```

clusterTime

```
clusterTime
```

Timestamp
clusterTime is the timestamp from the oplog entry associated with the event.

```
clusterTime
```

Due to oplog size limits, [multi-document transactions](https://www.mongodb.com/docs/manual/core/transactions/) may create multiple oplog entries. In a transaction, change stream events staged in a given oplog entry share the same clusterTime.

```
clusterTime
```

Events with the same clusterTime may not all relate to the same transaction. Some events don't relate to a transaction at all. Starting in MongoDB 8.0, this may be true for events on any deployment. In previous versions, this behavior was possible only for events on a sharded cluster.

```
clusterTime
```

To identify events for a single transaction, you can use the combination of lsid and txnNumber in the change stream event document.

```
lsid
```

```
txnNumber
```

Changed in version 8.0.
collectionUUID

```
collectionUUID
```

UUID
If the change occurred on a collection, collectionUUID indicates the collection's UUID. If the change occurred on a view, collectionUUID doesn't exist.

```
collectionUUID
```

```
collectionUUID
```

New in version 6.0.
lsid

```
lsid
```

document
The identifier for the session associated with the transaction.
Only present if the operation is part of a [multi-document transaction.](https://www.mongodb.com/docs/manual/core/transactions/)
ns

```
ns
```

document
The namespace (database and or collection) affected by the event.
ns.coll

```
ns.coll
```

string
The name of the collection where the event occurred.
ns.db

```
ns.db
```

string
The name of the database where the event occurred.
nsType

```
nsType
```

string
Shows the namespace type of the collection created by this event.

```
"collection"
```

```
"timeseries"
```

```
"view"
```

New in version 8.1: (also available in 8.0.5).
operationDescription

```
operationDescription
```

document
Additional information on the change operation.
This document and its subfields only appears when the change stream uses [expanded events.](https://www.mongodb.com/docs/manual/reference/change-events/)
New in version 6.0.

```
operationDescription.
```

```
idIndex
```

document
The default _id index for a new collection.

```
_id
```

New in version 6.0.
operationType

```
operationType
```

string
The type of operation that the change notification reports.
Returns a value of create for these change events.

```
create
```

txnNumber

```
txnNumber
```

NumberLong
Together with the [lsid](https://www.mongodb.com/docs/manual/reference/change-events/create/), a number that helps uniquely identify a transction.
Only present if the operation is part of a [multi-document transaction.](https://www.mongodb.com/docs/manual/core/transactions/)
wallTime

```
wallTime
```

[ISODate](https://www.mongodb.com/docs/manual/reference/glossary/)
The server date and time of the database operation. wallTime differs from clusterTime in that clusterTime is a timestamp taken from the oplog entry associated with the database operation event.

```
wallTime
```

```
clusterTime
```

```
clusterTime
```

New in version 6.0.
